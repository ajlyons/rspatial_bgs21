R-ArcGIS Bridge

The R-ArcGIS Bridge is a R package that allows you to exchange spatial data between R and ArcGIS Pro / ArcGIS.com.


Compatibility & Installation

Requires a licensed copy of ArcGIS Pro installed

Install the package from within ArcGIS Pro

  1. Close RStudio
  2. ArcGIS Pro >> Project >> Options >> Geoprocessing >> R-ArcGIS Support
  3. Open RStudio
  4. library(arcgisbinding)


Instructions on ArcMap Users available here.


What You Can Do with the R-ArcGIS Bridge

  • View spatial metadata before importing data into R.
  • Import spatial layers (both vector and raster) from geodatabases, Shapefiles, etc.
  • Create R scripts that you run from ArcGIS.

R-ArcGIS Bridge is primarily for transferring spatial data between R and ArcGIS.

It does not have all the functionality of arcpy, which allows you to run many ArcGIS Pro geoprocessing functions from Python.

Example: Import data from a Geodatabase

You don’t need to use the R-ArcGIS Bridge to read from a geodatabase. You can import layers from a geodatabase with sf::st_read().

You will however need the R-ArcGIS Bridge to to write to a geodatabase.

Step 1. Load arcgisbinding and verify the license

library(arcgisbinding)
library(sf)

## First verify the license
arc.check_product()
## product: ArcGIS Pro (12.6.1.24783)
## license: Advanced
## version: 1.0.1.244


Step 2. Create a ‘Connection Object’ to a feature class in a geodatabase

‘feature class’ = vector layer in a geodatabase
‘feature service’ = vector layer online

The second step is to create a ‘connection’ to a specific feature class (layer) in your geodatabase.

## Create a connect to the 'Rivers' feature class
yose_rivers_con <- arc.open(path="./data/yose_hydrology.gdb/Rivers")

## Display info (metadata)
yose_rivers_con
## dataset_type    : FeatureClass
## path            : ./data/yose_hydrology.gdb/Rivers

## WKID            : 26911

Connection objects allows you to ‘peak’ into a spatial layer without actually importing it.

If you get an error with arc.open(), try passing an absolute path name.


Step 3. Import the Feature Class into R as an “ArcGIS data frame”

Import the layer into R with arc.select():

yose_rivers_adf <- arc.select(yose_rivers_con)
class(yose_rivers_adf)
## [1] "arc.data"   "data.frame"

View properties of the ArcGIS data frame:

##  [1] "OBJECTID"   "ComID"      "FDate"      "Resolution" "GNIS_ID"    "GNIS_Name"  "LengthKM"  
##  [8] "ReachCode"  "FlowDir"    "WBAreaComI" "FType"      "FCode"      "Shape_Leng" "Enabled"   
## [15] "Shape_Le_1"

## WKID            : 26911

Step 4. Convert the ArcGIS Data Frame into a sf object

You can use arc.data2sf() to convert the ArcGIS data frame to sf:

yose_rivers_sf <- arc.data2sf(yose_rivers_adf)
class(yose_rivers_sf)
## [1] "sf"         "data.frame"

Plot to see how it looks:

plot(yose_rivers_sf %>% st_geometry(), axes=TRUE)

Writing to geodatabases

Write sf objects to a geodatabase with:

arc.write(path, data, overwrite)

Importing Data from ArcGIS.com

Option 1: Import the JSON

If the feature service is 1) public on ArcGIS.com (or an ArcGIS Entrprise Server), and 2) was published with JSON enabled, you can import it directly.

Look for a link to a JSON download Example.



Not all feature services have a JSON link. It depends on whether JSON sharing was enabled by person who published the layer.

If it doesn’t have a JSON link, you can still import the layer into R using the R-ArcGIS Bridge (see below).


Normally you can import GeoJSON files with st_read():

esri2sf

uc_props_url <- "https://services2.arcgis.com/wx8u046p68e0iGuj/arcgis/rest/services/UC_Properties/FeatureServer/0"

Importing AGOL Data with the R-ArcGIS Bridge

Whether or not a Feature Service has JSON downloading enabled, you can import it into R using the R-ArcGIS Brige.

The first step is to feed the reference URL into arc.open():

uc_props_url <- "https://services2.arcgis.com/wx8u046p68e0iGuj/arcgis/rest/services/UC_Properties/FeatureServer/0"

## Create a connect to the 'Rivers' feature class
uc_props_ftsrv_con <- arc.open(path = uc_props_url)

## Display info (metadata)
uc_props_ftsrv_con
## dataset_type    : FeatureClass
## path            : https://services2.arcgis.com/wx8u046p68e0iGuj/arcgis/rest/services/UC_Properties/FeatureServer/0

Preparing Spatial Data to Publish on AGOL

If you want to share your spatial layers on ArcGIS.com (or ArcGIS Enterprise Server), you have to select which format to save it in.

But file formats are not all the same! Some can be added to a web map, others not. Some can be made availble to download, some not. Details

GeoJSON is a good option, you can:

  • Display in Map Viewer
  • Publish as a hosted web layer
  • Download

Shapefile - ok option if you don’t care about field names

kml/kmz - ok option if you’re geometry is simple, have few or no attributes

GeoPackage - bad option (AGOL allows download only)
